# Talkat v1.0.0 ‚Äî Hardening & Feature Parity Plan

> **Versioning**: Ships as a single `1.0.0` release. No intermediate version bumps ‚Äî Talkat has no external users today, so the next published version goes straight from `0.2.0` to `1.0.0` when distribution (¬ß6) is ready. `pyproject.toml` is the single source of truth (the old `VERSION` file is gone as of ¬ß3.10).

## Context

Talkat reached 0.2.0 as a working voice-to-text dictation tool for Wayland Linux with a client‚Äìserver architecture (warm model over Unix socket), dual ASR backends (faster-whisper + Vosk), and active maintenance.

A side-by-side audit against [voxd](https://github.com/jakovius/voxd) ‚Äî the most-starred adjacent project (260‚òÖ) ‚Äî surfaced features worth borrowing and confirmed Talkat's architecture is sound. A fork survey on voxd found no community successor: upstream has been silent since 2025-10-22 (~8 months) with 9 open PRs and 15 open issues unattended; the two most-active forks (`deepbadger/voxd`, 13 commits ahead but 0 stars; `rdoiron/voxd-plus`, 24 commits ahead but dormant since 2025-12-31) are solo/private efforts, not community replacements.

This release closes Talkat's known rough edges, establishes CI/CD (none exists today), completes type coverage, and cherry-picks the highest-value features from voxd that don't bloat scope.

## Goals
- Eliminate the listed hacks (PID race, ALSA suppression, signal handling, hardcoded timeouts)
- Establish CI for tests, lint, type-check, and release artifacts
- Reach mypy-strict type coverage (config already strict-ish; just flip `disallow_untyped_defs`)
- Add optional, opt-in LLM post-processing ‚Äî the one voxd feature clearly worth borrowing
- Add multi-distro packaging (`.deb` / `.rpm`) via nfpm
- Surface configuration knobs that already work but aren't exposed (language, model)

## Non-Goals (Deferred to V2)
- GUI / tray / Qt anything (would add ~150 MB of deps for an inherently terminal-friendly tool)
- WebSocket / bidirectional streaming (current Unix-socket HTTP path is fine)
- Wake word detection (whole new dep tree ‚Äî Porcupine / OWW)
- Speaker diarization (requires a second model ‚Äî pyannote etc.)
- Per-language quality validation (V1 exposes `language`; V2 validates across corpora)
- Provider-specific LLM SDKs beyond a generic OpenAI-compatible client
- Switching ASR backend to whisper.cpp ‚Äî faster-whisper (CTranslate2) is comparable; rewrite not worth it

### 3. Hardening ‚Äî fix the listed hacks ‚úÖ

All ten items landed in one sweep. From the audit, in rough order of value:

- [x] **ALSA error suppression** ‚Äî replaced the ctypes-libasound hack with a `_suppress_native_stderr()` context manager (fd-level `os.dup2` to `/dev/null`) wrapped around the PyAudio init + `stream.open()` calls in `AudioSession.__enter__` and `calibrate_microphone`. Works on musl/non-glibc, silences JACK + PortAudio noise too, and Python exceptions still propagate normally.
- [x] **PID-file race** ‚Äî option (b) `flock` refactor landed.
    - Added a `ProcessManager.locked(try_only=...)` context manager as the lock primitive; rewrote `toggle()` to take a start callback that runs under the lock; cli.py `listen` now writes its PID inside the lock before releasing it and calling `listen_once`, so a concurrent `talkat listen` cannot race in.
    - Fixed the Popen ‚Üí write_pid non-atomicity: `write_pid` raises `PIDWriteError` on I/O failure, and `start_background_process` kills the child if the PID couldn't be recorded.
    - Added `--try-lock` to `listen`, `start-long`, `stop-long`, `toggle-long`. For listen it both skips the retry and refuses to toggle off a running session ("already running" instead of stop).
    - Options (a) ‚Äî long-running CLI helper daemon with IPC ‚Äî and (c) ‚Äî systemd path/socket activation ‚Äî were considered and deferred:
      - (a) is the cleanest race fix and unlocks cross-mode coordination, but ~12‚Äì15h and adds a daemon + IPC protocol we don't yet need use cases for. Becomes attractive at v1.1+ if the project adds GUI/tray, real-time state queries, or multi-mode interactions.
      - (c) leans on systemd's job queue and deletes most of `process_manager.py`, but ~8‚Äì12h, shifts UX toward `systemctl` semantics, adds 50‚Äì200ms per invocation, and bets hard on systemd in user sessions. Reasonable v1.x direction if we want systemd to fully own management.
- [x] **Signal handling** ‚Äî the SIGINT/SIGTERM handler in `_set_stop_event_on_signal` now both sets the stop Event and raises `KeyboardInterrupt`. Without this, PEP 475 silently retries the EINTR'd `httpx.post` until the 120 s `http_timeout` finally expires. Raising propagates out via Python's standard exception path (same mechanism as the interpreter's default SIGINT handler), so the blocking call returns immediately and `listen_once` / `listen_continuous` shut down cleanly. (`signal.set_wakeup_fd()` was the original sketch in the plan, but it only helps selector-based loops ‚Äî sync httpx has no selector we can wake. Raise-from-handler is the equivalent fix for sync I/O.)
- [x] **Configurable timeouts via CLI flags** ‚Äî added `--max-recording` (listen only), `--silence-duration`, `--http-timeout` on `listen` and `long`. cli.py collects non-None values and passes a `config_overrides` dict through to `listen_once` / `listen_continuous`, which merges them on top of the file config before constructing the client + session.
- [x] **File upload size limit** ‚Äî server reads `max_upload_size_mb` from config (default 100 MB) and sets Flask `MAX_CONTENT_LENGTH` in `main()`; added a 413 error handler that returns a JSON body so clients can show the friendly limit. Client-side `file_processor.py` now stat()s the file before uploading and raises with a useful message rather than spending bandwidth on a doomed request.
- [x] **Long-mode transcript memory bound** ‚Äî dropped the `full_transcript: list[str]` accumulator. Each utterance is still written straight to the transcript file (which it already was); the final clipboard copy now reads that file back, so peak memory is independent of session length.
- [x] **Server model warm-up** ‚Äî `ModelService._warm_up()` runs a 0.5s silence frame through `model.transcribe` (faster-whisper) or a `KaldiRecognizer` (vosk) at the end of `initialize()`, before `waitress.serve` starts accepting requests. Failures log a warning but never block startup.
- [x] **Circuit breaker for long-mode** ‚Äî `listen_continuous` counts consecutive `TranscriptionServerError`s and aborts with a notification after `long_mode_max_consecutive_errors` (default 5). Successful transcribes reset the counter. `TranscriptionUnreachable` still trips out immediately (one-strike) as before.
- [x] **Systemd user-service hardening** ‚Äî the packaged `talkat.service` already had most of this; ported the same flags into `service.py._service_unit()` (the local-install path) and added `ProtectHome=read-only` to both. `PrivateDevices` intentionally left off (microphone access).
- [x] **`VERSION` vs `pyproject.toml` drift** ‚Äî deleted the stale `VERSION` file; `pyproject.toml` is the sole source of truth. Setup, PKGBUILD, and CI all already derived the version from package metadata or git tags, so nothing referenced the file in code.

**Done 2026-06-14.**

**Estimate**: ~25‚Äì30h ‚Üí actual ~3h.

### 4. Testing ‚úÖ
Existing tests cover config, process_manager, security, service ‚Äî solid but thin. Bring coverage up where it matters.

- [x] **Integration test** ‚Äî `tests/test_integration_file_processor.py` runs a real Flask + waitress server on a UDS and drives a fixture WAV through `transcribe_audio_file()` over the real httpx unix-socket transport. Covers the ¬ß3 max-upload 413 path, the client-side size guard, the unreachable-server abort, and an end-to-end transcript round-trip. The "typing call" half is in `tests/test_listen.py` (six tests) which stubs the transcription client and asserts that `safe_subprocess_run(["ydotool", "type", ...])` fires with sanitized text on success, is skipped on empty / unreachable / output-file paths.
- [x] **VAD unit tests** ‚Äî `tests/test_vad.py` exercises `AudioSession` with synthetic int16 chunks behind a stubbed PyAudio: no-VAD passthrough, pure-silence yields nothing, pre-roll inclusion, silence-duration cutoff, max-duration cap, stop_event short-circuit, resource cleanup on `__exit__`, no-mic raises. - [x] **Process manager** ‚Äî `tests/test_process_manager.py` now has 28 tests covering the ¬ß3 rewrite: `LockTimeout`/`PIDWriteError` class identity, `locked()` context manager (try_only fast-fail timing under 250 ms), stale-PID recovery (non-talkat process / garbage / empty file), `write_pid` atomic swap + tmp cleanup + `PIDWriteError` on unwritable dir, `toggle()` start/stop dispatch + rapid-fire alternation, `start_background_process` killing the child on `write_pid` failure (verified by capturing the Popen), and a real-child SIGINT happy-path for `stop_process`.
- [x] **Long-mode** ‚Äî `tests/test_long_mode.py` drives `listen_continuous` with a programmable `FakeTranscriber`. ### 6. Distribution
- [ ] nfpm config (`packaging/nfpm.yaml`) + release workflow produces `.deb` and `.rpm`
- [ ] README install section gets apt / dnf instructions
- [ ] AUR + `setup.sh` paths keep working unchanged (the AUR PKGBUILD lives in the separate AUR repo per CLAUDE.md)
- [ ] PyPI release deferred ‚Äî too many system-binary deps (`pyaudio`, `ydotool`) to be a clean pip install
- [ ] **Layered config merge** ‚Äî `load_app_config()` currently overlays `CODE_DEFAULTS` with `~/.config/talkat/config.json` if it exists, *otherwise* `/etc/talkat/config.json`. A user config wholesale shadows a system config, so a sysadmin-shipped `/etc/talkat/config.json` (which is how nfpm-packaged installs will deliver org-wide defaults) disappears the moment a user creates their own. Fix: merge in the order `CODE_DEFAULTS ‚Üí /etc/talkat/config.json ‚Üí ~/.config/talkat/config.json ‚Üí CLI overrides`, each layer partially overriding the previous. Small (~15 lines in `config.py` + `paths.py`). Also fix the misleading `get_config_file()` docstring ("checking system-wide first" ‚Äî the code is user-first). Bundling here because the layered semantics only start mattering once we actually ship a `/etc` default in the deb/rpm.
- [ ] **Extend CLI overrides to `file` / `batch` / `server`?** ‚Äî ¬ß3.4 added `--max-recording` / `--silence-duration` / `--http-timeout` on `listen` + `long` only. Once nfpm packaging is real, decide whether `file --http-timeout`, `server --max-upload-size`, etc. are worth surfacing or whether the config file + env vars are enough.

**Estimate**: ~8h ‚Üí ~10h with the config-merge work

## V2 / Deferred ‚Äî Documented for Future

| Item | Why deferred |
|---|---|
| GUI / tray (PyQt6 or GTK) | Adds ~150 MB of GUI deps for an inherently terminal-friendly tool; mode-switch via CLI is enough |
| Multi-language **validation** | Whisper supports many languages; quality work is per-language threshold tuning + corpus testing, real but doesn't gate v1.0.0 |
| WebSocket / bidirectional streaming | The current Unix-socket HTTP path is fast enough; WS only matters for sub-second feedback we don't promise |
| Wake word detection | Pulls in Porcupine / openWakeWord ‚Äî whole new dep tree, separate model |
| Speaker diarization | Requires a second model (pyannote etc.); meaningful new scope |
| Provider-specific LLM clients (Anthropic SDK, xAI SDK, Gemini SDK) | Each adds a dep tree; the generic OpenAI-compatible endpoint covers most local + cloud cases for v1.0.0 |
| Mobile companion / cloud sync | Not aligned with offline-first design |
| Replace ASR backend with whisper.cpp | voxd uses it; faster-whisper (CTranslate2) is comparable; not worth the rewrite |
| aarch64 packaging | Low signal-to-noise for our audience right now; add when there's demand |

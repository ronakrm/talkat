# Talkat v1.0.0 — Hardening & Feature Parity Plan

> **Versioning**: Ships as a single `1.0.0` release. No intermediate version bumps — Talkat has no external users today, so the next published version goes straight from `0.2.0` to `1.0.0` when distribution (§6) is ready. The `VERSION` file and `pyproject.toml` both get set to `1.0.0` as part of §3.

## Context

Talkat reached 0.2.0 as a working voice-to-text dictation tool for Wayland Linux with a client–server architecture (warm model over Unix socket), dual ASR backends (faster-whisper + Vosk), and active maintenance.

A side-by-side audit against [voxd](https://github.com/jakovius/voxd) — the most-starred adjacent project (260★) — surfaced features worth borrowing and confirmed Talkat's architecture is sound. A fork survey on voxd found no community successor: upstream has been silent since 2025-10-22 (~8 months) with 9 open PRs and 15 open issues unattended; the two most-active forks (`deepbadger/voxd`, 13 commits ahead but 0 stars; `rdoiron/voxd-plus`, 24 commits ahead but dormant since 2025-12-31) are solo/private efforts, not community replacements.

This release closes Talkat's known rough edges, establishes CI/CD (none exists today), completes type coverage, and cherry-picks the highest-value features from voxd that don't bloat scope.

## Goals
- Eliminate the listed hacks (PID race, ALSA suppression, signal handling, hardcoded timeouts)
- Establish CI for tests, lint, type-check, and release artifacts
- Reach mypy-strict type coverage (config already strict-ish; just flip `disallow_untyped_defs`)
- Add optional, opt-in LLM post-processing — the one voxd feature clearly worth borrowing
- Add multi-distro packaging (`.deb` / `.rpm`) via nfpm
- Surface configuration knobs that already work but aren't exposed (language, model)

## Non-Goals (Deferred to V2)
- GUI / tray / Qt anything (would add ~150 MB of deps for an inherently terminal-friendly tool)
- WebSocket / bidirectional streaming (current Unix-socket HTTP path is fine)
- Wake word detection (whole new dep tree — Porcupine / OWW)
- Speaker diarization (requires a second model — pyannote etc.)
- Per-language quality validation (V1 exposes `language`; V2 validates across corpora)
- Provider-specific LLM SDKs beyond a generic OpenAI-compatible client
- Switching ASR backend to whisper.cpp — faster-whisper (CTranslate2) is comparable; rewrite not worth it

## Workstreams

### 1. CI/CD foundation
Highest priority — unlocks enforced quality on everything else. There is no `.github/` directory today; this is greenfield.

- [ ] `.github/workflows/test.yml` — pytest on push/PR, matrix on Python 3.11 + 3.12
- [ ] `.github/workflows/lint.yml` — `ruff format --check` + `ruff check`
- [ ] `.github/workflows/typecheck.yml` — `mypy` (eventually strict)
- [ ] `.github/workflows/release.yml` — tag push → build `.deb` / `.rpm` via nfpm → GitHub release
    - x86_64 only in v1.0.0 (aarch64 deferred — low signal-to-noise for our audience)
- [ ] Optional: coverage upload to Codecov, badge in README
- [ ] Note: pre-commit already configured in `.pre-commit-config.yaml`; CI just enforces what local hooks already check

**Estimate**: ~10h

### 2. Type safety
Bring the codebase to mypy-strict. The configuration in `pyproject.toml` is already most of the way there — `warn_return_any`, `check_untyped_defs`, `no_implicit_optional`, etc. are on. Just need to flip `disallow_untyped_defs = true` and fix the fallout.

- [ ] Audit + fix `model_server.py` (Flask routes return untyped JSON today — biggest gap)
- [ ] Audit + fix `main.py` and `cli.py`
- [ ] Flip `disallow_untyped_defs = true` in `pyproject.toml`
- [ ] Resolve remaining violations
- [ ] Add `py.typed` marker so downstream consumers see types

**Estimate**: ~10h

### 3. Hardening — fix the listed hacks

From the audit, in rough order of value:

- [ ] **ALSA error suppression** — replace the ctypes-libasound hack in `record.py:19-37` with a clean stderr redirect (`os.dup2`) at the process level. Brittleness goes away; works on non-glibc systems.
- [ ] **PID-file race** — pick one approach **before coding**:
    - (a) Single long-running CLI helper that owns toggles (cleanest, most refactor)
    - (b) Atomic compare-and-swap via `flock` on a control file (least refactor)
    - (c) Systemd path-based activation (relies on the user unit)
    - Recommend (b) for v1.0.0; (a) is a v1.1 architectural change.
- [ ] **Signal handling** — adopt `signal.set_wakeup_fd()` so a SIGINT during a blocking `httpx.post` interrupts the call rather than waiting on the 120 s timeout
- [ ] **Configurable timeouts via CLI flags** — `--max-recording`, `--silence-duration`, `--http-timeout`. Currently config-file-only.
- [ ] **File upload size limit** — explicit Flask `MAX_CONTENT_LENGTH` + friendly client-side check in `file_processor.py`
- [ ] **Long-mode transcript memory bound** — flush to disk every N segments rather than accumulating in memory (currently grows unbounded)
- [ ] **Server model warm-up** — preload + run one dummy frame on startup before accepting requests (first-request latency disappears)
- [ ] **Circuit breaker for long-mode** — if the server returns connection errors N times in a row, exit cleanly with a notification rather than spamming retries forever
- [ ] **Systemd user-service hardening** — add `NoNewPrivileges=true`, `ProtectSystem=strict`, `ProtectHome=read-only`, `PrivateTmp=true`, `RestrictNamespaces=true`. User service, but defence-in-depth.
- [ ] **`VERSION` vs `pyproject.toml` drift** — `VERSION` says `0.1.0`, `pyproject.toml` says `0.2.0`. Pick one source of truth (recommend `pyproject.toml`) and either delete `VERSION` or generate it from the package metadata. Both land on `1.0.0` at the §6 release cut.

**Estimate**: ~25–30h

### 4. Testing
Existing tests cover config, process_manager, security, service — solid but thin. Bring coverage up where it matters.

- [ ] **Integration test** — fixture WAV → mock model server (or real one with a tiny model) → assert transcript + typing call
- [ ] **VAD unit tests** — synthetic audio with known speech segments → assert frames yielded, pre-roll, silence detection
- [ ] **Process manager** — expand: lock contention, stale-PID recovery, atomic swap, rapid-fire toggle
- [ ] **Long-mode** — fake server emitting transcripts; assert client behaviour on disconnect, on memory cap, on circuit-breaker trip
- [ ] Coverage target ≥75% on core modules; report via `pytest-cov` (already a dev dep)

**Estimate**: ~25h

### 5. Easy feature wins (cherry-picked from voxd)

#### 5a. AIPP-style post-processing (optional, opt-in)
Pipe transcripts through an LLM before they hit `ydotool`. Genuinely useful (e.g., "fix grammar", "format as bullet list", "rewrite as Python"). voxd's only clear unique-feature win.

- [ ] Pluggable post-processor pipeline in `main.py` between transcription and typing
- [ ] Built-in providers in v1.0.0:
    - **Ollama** (local) — fits the offline-first ethos
    - **Generic OpenAI-compatible HTTP endpoint** (covers OpenRouter, LM Studio, llama.cpp server, OpenAI itself, and most hosted offerings)
- [ ] Config schema: ordered list of `{provider, model, prompt}` steps; users can chain
- [ ] CLI: `talkat listen --postprocess <profile-name>`; profiles defined in config
- [ ] Graceful fall-through: if LLM is unreachable or exceeds timeout, type the raw transcript with a notification
- [ ] **No new heavy deps** — uses the `httpx` already in our dep tree

**Estimate**: ~12h

#### 5b. Surface language config
Already supported by Whisper; just expose cleanly.

- [ ] Add `language` to default config + `--language` CLI flag
- [ ] Document in README
- [ ] Per-language calibration / quality validation deferred to V2 (this is just plumbing)

**Estimate**: ~2h

#### 5c. Model management CLI
voxd's `voxd-model` is handy — Whisper users juggle multiple model sizes (tiny / base / small / medium / large).

- [ ] `talkat model list` — show installed models + sizes
- [ ] `talkat model download <name>` — pull a faster-whisper model variant via the HF cache it already uses
- [ ] `talkat model use <name>` — set default in config
- [ ] No model-fetcher daemon needed; faster-whisper handles downloads

**Estimate**: ~6h

### 6. Distribution
- [ ] nfpm config (`packaging/nfpm.yaml`) + release workflow produces `.deb` and `.rpm`
- [ ] README install section gets apt / dnf instructions
- [ ] AUR + `setup.sh` paths keep working unchanged (the AUR PKGBUILD lives in the separate AUR repo per CLAUDE.md)
- [ ] PyPI release deferred — too many system-binary deps (`pyaudio`, `ydotool`) to be a clean pip install

**Estimate**: ~8h

## V2 / Deferred — Documented for Future

| Item | Why deferred |
|---|---|
| GUI / tray (PyQt6 or GTK) | Adds ~150 MB of GUI deps for an inherently terminal-friendly tool; mode-switch via CLI is enough |
| Multi-language **validation** | Whisper supports many languages; quality work is per-language threshold tuning + corpus testing, real but doesn't gate v1.0.0 |
| WebSocket / bidirectional streaming | The current Unix-socket HTTP path is fast enough; WS only matters for sub-second feedback we don't promise |
| Wake word detection | Pulls in Porcupine / openWakeWord — whole new dep tree, separate model |
| Speaker diarization | Requires a second model (pyannote etc.); meaningful new scope |
| Provider-specific LLM clients (Anthropic SDK, xAI SDK, Gemini SDK) | Each adds a dep tree; the generic OpenAI-compatible endpoint covers most local + cloud cases for v1.0.0 |
| Mobile companion / cloud sync | Not aligned with offline-first design |
| Replace ASR backend with whisper.cpp | voxd uses it; faster-whisper (CTranslate2) is comparable; not worth the rewrite |
| aarch64 packaging | Low signal-to-noise for our audience right now; add when there's demand |

## Sequencing

1. **Week 1** — CI/CD foundation (§1). Everything else lands through CI from day one.
2. **Week 1–2** — Type safety (§2) + testing (§4) in parallel. Each surfaces bugs that feed §3.
3. **Week 2–3** — Hardening sweep (§3). Decide PID-race approach before writing code.
4. **Week 3–4** — Feature additions (§5). AIPP is the biggest piece; language + model-CLI are small.
5. **Week 4** — Distribution (§6). nfpm packaging + first tagged release through the new pipeline. **This is the `1.0.0` cut** — bump `pyproject.toml` + `VERSION`, tag `v1.0.0`, release job builds and publishes.
